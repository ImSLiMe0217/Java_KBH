## 0601 진도

### 1. 객체지향프로그래밍
- 핵심 지향점
  - 프로그램의 부품화: 데이터와 이를 처리하는 메서드를 객체라는 하나의 단위로 묶어 관리
  - 재사용성과 유지보수성이 높음

#### 1.1 객체, 클래스, 인스턴스
- 객체: 유무형의 모든 실체
  - 고유한 상태와 동작을 가짐
  - 상태 -> 필드, 동작 -> 메소드
- 클래스: 객체를 생성하기 위한 설계도
  - 객체의 형태와 기능을 자바 코드로 사전에 명시해둔 규격, 메모리에 실체는 없음
- 인스턴스: 클래스라는 설계도를 바탕으로 'new'연산자를 사용하여 힙 메모리에 생성한 실체

#### 1.2 클래스의 구성요소
- 필드: 객체의 상태, 속성을 정의한 변수
- 생성자: 객체가 new로 생성될 때 초기 세팅을 담당
- 메서드: 객체의 동작, 행위, 기능을 담고있는 코드 블럭

```java
public class 클래스명 {
    // 필드
    [접근제어자] 타입 변수명;
    
    // 생성자 (Python 기준 init메소드랑 같은 역할)
    // 없어도 됨(자바가 기본생성자를 만들어줌), 오버로딩 가능
    [접근제어자] 클래스명(매개변수타입 매개변수명, ...) {
        // 초기 세팅코드
    }
    
    // 메서드
    [접근제어자] 반환타입 메소드명 (매개변수타입 매개변수명, ...) {
        // 메소드 내부 코드
    }
    ...
}
```

#### 1.3 생성자 선언 시 주의할 점
- 생성자의 매개변수는 생성자 내에서만 사용가능한 `지역변수`
- 필드에서 선언한 변수는 클래스 내의 모든 메소드에서 사용 가능한 `전역변수`
- `지역변수`와 `전역변수`의 식별자가 동일할 경우 호출 시에는 선언한 구역의 `지역변수`가 우선됨
- 예시:

```java
public class ValPosTest {
    // 필드
    int a = 10;
    
    void test() {
        int a = 20;
        System.out.println(a); // 20
    }
}
```

- 생성자의 매개변수와 필드에 선언된 변수의 식별자를 동일하게 사용하고 싶은 경우 `this.식별자` 형식을 사용함
- 예시:

```java
public class 클래스명 {
    // 필드
    int a;
    
    클래스명(int a) {
        this.a = a; 
        // this.a -> 필드에 선언된 a
        // a -> 지역변수 a
    }
}
```

### 1.4 캡슐화와 접근제어자
- 캡슐화: 중요한 데이터와 이를 처리하는 메서드를 하나로 묶고, 외부에서 임의로 변경하지 못하도록 데이터를 은닉하는 기법
- 접근제어자: private < default < protected < public 순으로 접근 권한을 제한
  - public: 어디에서든 접근 가능
  - default: 제어자를 생략하면 자동으로 부여, 동일 패키지 내부에서만 접근을 허용
  - private: 선언된 해당 클래스 내부에서만 호출 및 접근 가능
  - (protect는 상속 단원에서 다시 설명)

### 1.5 정적(static) 멤버

- static 키워드가 붙은 필드나 메서드는 인스턴스 생서을 하지 않아도 클래스명으로 바로 접근 가능
- 모든 인스턴스가 공유하는 메모리에 저장됨
- static 메서드의 제한 사항:
  - this 키워드는 static메서드 내부에서 사용할 수 없음
  - static 메서드 내부에서는 오직 static 필드(클래스 변수)나 static 메서드만 직접 호출 가능
  
```java
public class Test {
    static void staticMethod() {
        System.out.println("static 메소드 호출");
    }
    
    void instanceMethod() {
        System.out.println("instance 메소드 호출");
    }
    
    static void main() {
        staticMethod();
//        instanceMethod(); -> 컴파일에러 발생
        
        
        Test test = new Test();     // 동일 클래스를 new로 생성
        test.instanceMethod();      // 이런 방식으로는 호출 가능
  }
}
```
