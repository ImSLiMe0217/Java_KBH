## 0526 진도

### 1. 배열

#### 1.1 기본 형태

- 같은 타입의 데이터 여러 개를 하나로 묶어서 순차적으로 관리하는 자료구조
- index를 이용하여 배열의 각 요소를 참조
- 특징:
    - 같은 타입의 데이터만 저장 가능
    - 고정된 크기: 선언 및 생성할 때 크기를 지정, 이후 변경 불가능
    - 연속된 메모리 공간
    - 참조형 데이터

```java
void main() {
    // 선언, 생성, 초기화를 동시에 수행
    int[] array1 = new int[] {0, 1, 2, 3, 4};

    // 선언과 생성만 수행 -> 기본값 0으로 초기화 됨
    int[] array2 = new int[5];
    array2[1] = 100;  // [인덱스]를 통해 내부 값에 접근


    int array3[]; // 이렇게 선언하는 것도 가능하나 권장되지 않음
}
```

- 배열의 생성과 초기화를 동시에 수행하면 [ ]안에 크기를 지정할 필요 없음
- 초기화를 따로 할 경우 [ ]안에 크기를 지정해주어야 함

#### 1.2 배열의 크기와 인덱스

- 크기
    - 배열에 저장 가능한 전체 요소의 개수
    - `(배열이름).length`: 배열의 크기를 확인
- 인덱스
    - 인덱스는 0부터 시작

### 2. 메모리 구조

#### 2.1 Stack 영역

- 메소드 호출 시 사용되는 로컬 변수들이 적재되는 영역
- 기본 타입 변수는 실제 값을 stack영역에 저장함
- 참조 타입 변수는 값이 저장된 Heap영역 메모리의 주소를 저장함

#### 2.2 Heap 영역

- new 연산자로 생성된 모든 인스턴스와 배열 객체가 할당되는 영역
- Garbage Collector에 의해 참조 관계가 끊어진 객체들이 감지되어 자동으로 수거됨
