### 컴파일과 인터프리팅

1. 컴파일
    - 컴파일러를 통해 수행
    - 모든 소스코드를 한 번에 번역
    - 사용자 친화적
      - 개발단계에서 각 운영체제에 맞는 언어로 번역해서 배포
      - 실행속도가 빠름
      - 개발과정이 오래 걸림
    - 대표적인 컴파일러 언어
      - C / C++, Rust, Golang
    

2. 인터프리팅
    - 인터프리터를 통해 수행
    - 소스 코드를 한 줄씩 즉석에서 번역 후 즉시 결과 실행
    - 개발자 친화적
      - 사용자가 인터프리터를 설치해서 사용해야함
      - 개발과정이 빠름
      - 실행속도가 상대적으로 느림
    - 대표적인 인터프리터 언어
      - Python, JavaScript, Ruby


3. 하이브리드 언어
   - 소스코드를 바이트코드(중간어)로 1차 컴파일
   - 실행환경에서 가상머신(VM)을 통해 기계어로 번역
   - 컴파일 언어와 인터프리터 언어의 단점을 보완
   - 대표적인 하이브리드 언어
     - Java, C#

### 절차지향과 객체지향 프로그래밍

1. 절차지향 프로그래밍
   - 프로그램의 실행 순서를 중시
   - 소스코드 작성 시 함수를 중심으로 작성
   - 특징
     - 컴퓨터의 처리구조와 유사, 실행속도가 빠름
     - 프로그램의 흐름을 파악하기 쉬움
     - 데이터와 함수가 분리되어 있어 데이터 구조 변경시 관련 함수를 모두 수정해야 함
   - 대표 언어
     - C

   
2. 객체지향 프로그래밍
   - 데이터와 처리함수를 '객체'라는 하나의 단위로 묶어 관리
   - 실제 세계의 사물을 모델링하여 설계
   - 특징
     - 코드 재사용성이 높고 유지보수가 쉬움
     - 설계 단계에서 시간이 많이 소요됨
     - 실행속도가 절차지향에 비해 상대적으로 느림
   - 대표 언어
     - Java, C++, Python, C#
