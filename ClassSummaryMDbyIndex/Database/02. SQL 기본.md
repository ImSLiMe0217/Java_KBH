# SQL 기본

- SQL(Structured Query Language)
- 관계형 데이터베이스(RDBMS)의 데이터를 정의, 조작, 제어하기 위해 고안된 표준 컴퓨터 언어
- 원하는 결과만 명시하여 명령을 수행하는 비절차적 언어
- DBMS에 직접 명령을 전달하여 테이블 스키마를 설계, 레코드를 검색, 추가, 갱신 삭제 가능
- 권한 제어 및 트랜잭션 관리도 수행

___

## 1. SQL의 주요 기능별 분류

### DDL(Data Definition Language): 데이터 정의어

- 데이터베이스의 물리적인 테이블 구조를 생성, 변경, 삭제하는 명령어 집합
- 관계형 데이터베이스의 뼈대가 되는 테이블을 정의할 때 사용

#### 데이터베이스 생성, 전환 및 삭제

- 생성: 신규 데이터베이스 생성

```mysql
CREATE DATABASE board_db;
```

- 전환: 이후 실행할 명령어들이 적용될 대상을 지정

```mysql
USE board_db;
```

- 삭제: 데이터베이스 삭제 (내부의 모든 테이블과 데이터 영구적으로 소멸)

```mysql
DROP DATABASE board_db;
```

#### 테이블 생성

- 데이터베이스 내의 데이터를 구조화하여 저장할 수 있도록 물리적인 테이블 구조를 정의하고 생성
- 컬럼의 이름, 데이터 타입 및 다양한 무결성 제약조건을 사전에 규정
- 예시:

```mysql
CREATE TABLE member -- 테이블이름
(
    -- 열_이름 데이터타입 세부조건들
    id         INT AUTO_INCREMENT PRIMARY KEY,
    email      VARCHAR(100) NOT NULL UNIQUE,
    password   VARCHAR(255) NOT NULL,
    name       VARCHAR(50)  NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 주요 데이터 타입 종류

- 숫자형
    - `INT`: 정수형 데이터
    - `BIGINT`: 대용량 정수형 데이터
    - `DECIMAL`: 고정 소수점 실수
- 문자형
    - `CHAR(길이)`: 고정 길이 문자열 데이터
    - `VARCHAR(길이)`: 가변 길이 문자열 데이터
    - `TEXT`: 본문글, 댓글 내용 등 대용량 텍스트 데이터
- 날짜/시간형
    - `DATE`: 연, 월, 일 날짜 정보만 저장(YYYY-MM-DD 형식)
    - `TIME`: 시, 분, 초 시간 정보만 저장(HH:MM:SS 형식)
    - `DATETIME`
        - 타입존에 영향을 받지 않고 입력된 고정 시간값을 보존
        - 2038년 표현 한계가 없고 타임존 설정 불일치로 인한 오차를 방지 가능
    - `TIMESTAMP`
        - 시스템의 타임존에 따라 내부적으로 UTC 시간값으로 자동 변환되어 저장 및 조회

#### 주요 제약조건

- `PRIMARY KEY` (기본 키)
    - 테이블 내의 행을 고유하게 식별할 수 있는 컬럼으로 지정
    - 중복값과 빈 값(null)을 허용하지 않음
- `FORIEIGN KEY` (외래 키)
    - 다른 테이블의 기본 키를 참조하여 테이블 간의 연결 관계를 형성
    - 데이터 참조 무결성을 보장
    - 참조 옵션 종류
        - `ON DELETE CASCADE`: 부모 테이블의 레코드가 삭제되면 연관된 자식 테이블의 레코드도 자동 삭제
        - `ON DELETE SET NULL`: 부모 테이블의 레코드가 삭제되면 자식 테이블의 외래 키 값을 NULL로 변경
        - `ON DELTE RESTRICT`: 자식 테이블이 참조하고있는 부모 테이블의 레코드는 삭제가 불가능하도록 차단
    - 외래 키 지정 방법

```mysql
-- 컬럼 레벨 정의
-- [컬럼] [데이터_타입] REFERENCES [부모_테이블(부모 컬럼)]
user_id INT REFERENCES member(id)

--- 테이블 레벨 정의
-- FOREIGN KEY (컬럼) REFERENCES [부모테이블(부모컬럼)]
FOREIGN KEY (user_id) REFERENCES member(id)

-- 테이블 생성 후 추가 정의
-- ALTER TABLE [테이블명] ADD CONSTRAINT [fk_post_member] FOREIGN KEY (컬럼 이름) REFERENCES [부모테이블(부모컬럼)] 
ALTER TABLE post
    ADD CONSTRAINT fk_post_member FOREIGN KEY (user_id) REFERENCES member (id);
```

- `NOT NULL`
    - 해당 컬럼에 반 값을 허용하지 않음
- `UNIQUE`
    - 컬럼 내의 모든 데이터 값이 서로 중복되지 않음
- `DEFAULT`
    - 데이터를 저장할 때 값을 명시하지 않으면 자동으로 입력될 기본값을 설정
    - 기본값으로 사용할 수 있는 주요 유형
        - 리터럴 상수: 정수형 기본값(0), 문자형 기본값('N', 'USER'), 빈문자열('') 등을 사용
        - NULL: 값을 별도로 지정하지 않았을 때 빈 상태로 기록되도록 허용
        - 내장 함수: 날짜 및 시간 형식에서 시스템의 현재 시각을 채우는 CURRENT_TIMESTAMP 등을 사용
- `AUTO_INCREMENT`
    - 새로운 행이 추가될 때마다 정수형 식별자 값을 자동으로 1씩 증가하여 부여

#### 테이블 수정(ALTER TABLE)

- 생성된 테이블의 구조를 변셩
- 기존 테이블의 컬럼 추가 및 제거, 컬럼의 데이터 타입 및 제약조건을 수정

```mysql
-- 컬럼 추가
ALTER TABLE 테이블명
    ADD 컬럼명 데이터타입 [제약조건];

-- 컬럼 수정
ALTER TABLE 테이블명
    MODIFY 컬럼명 데이터타입 [제약조건];

-- 컬럼 삭제
ALTER TABLE 테이블명
    DROP COLUMN 컬럼명;
```

#### 테이블 제거(DROP TABLE)

- 더 이상 사용하지 않는 테이블을 영구적으로 제거
- 테이블 구조및 내부 데이터 소멸

```mysql
DROP TABLE 테이블명;
```

#### 테이블 초기화(TRUNCATE TABLE)

- 테이블의 구조는 남겨둔 채 내부의 모든 데이터행을 초기 상태로 리셋
- DELETE와 달리 로그를 남기지 않아 롤백이 불가능
- 대용량 데이터를 빠르고 효율적으로 비울 때 주로 활용

```mysql
TRUNCATE TABLE 테이블명;
```
