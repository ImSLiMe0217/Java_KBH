## 복습

### 1. 다차원 배열

- 배열의 요소로 다른 배열들을 가지고 있는 배열
- 요소로 가지고 있는 배열들의 크기는 서로 달라도 됨

```java
void main() {
    int[][] arr2d1 = {{1, 2, 3}, {4, 5, 6}}; // 2 x 3 행렬

    int[][] arr2d2 = {{1, 2}, {3, 4, 5, 6}};
}
```

### 2. 향상된 for문

- 배열의 내부 원소들을 모두 순회할 때 사용하는 for문
- 파이썬을 예시로 들면 for (i in list)같은 형태라 보면 됨

```java
void main() {
    int[] arr = {1, 2, 3, 4, 5};

    for (int element : arr) {
        // element -> 배열의 요소를 저장할 변수
        // arr -> 순회할 배열

        System.out.println(element); // 1 ~ 5까지 순차적으로 출력
    }
}
```

### 3. 얕은 복사, 깊은 복사

- 얕은 복사
    - 원본배열과 복사한 배열이 서로 연동되어 있음
    - 한 쪽에서 값을 변경하면 다른 한 쪽에서도 값이 반영됨
    - `int[] copy = origin` 형태로 복사가능
- 깊은 복사
    - 원본배열과 복사한 배열이 서로 다른 배열로 취급됨
    - 한 쪽에서 값을 변경하더라도 다른 한 쪽에서는 반영되지 않음
    - for문을 사용하거나 `System.arrayCopy()`, `Arrays.copyOf()`, `원본배열.clone()`등을 사용해서 복사가능

### 4. Arrays 라이브러리 클래스

- 배열 객체를 손쉽게 다루기 위한 메소드를 모아놓은 클래스
- 주요 기능
    - Arrays.toString(): 배열 객체를 문자열로 바꿔주는 메소드
    - Arrays.sort(): 배열의 내부 요소를 오름차순으로 정렬해주는 메소드
    - Arrays.copyOf(): 배열을 깊은 복사할 때 사용하는 메소드
    - Arrays.equals(): 두 배열의 요소들의 값과 순서가 모두 동일한지 확인하는 메소드

### 2. 메소드

- 특정 기능을 수행하기 위해 관련된 코드들을 하나의 블록으로 묶어놓은 것
- 필요할 때마다 호출해서 여러번 재사용 가능
- 장점
    - 코드의 재사용성 증대
    - 코드의 중복 제거
    - 유지보수성 향상
    - 가동성 및 구조화

#### 2.1 메소드 선언

- 형식:

```
[접근제어자] 반환타입 메서드명(매개변수타입 매게변수명) {
    본문
    return 반환값:
        }
```

-
    - 접근제어자: 메서드를 호출할 수 있는 범위를 지정
        - public: 어디서든 호출 가능
        - private: 같은 클래스 내부에서만 호출가능
        - protected: 동일한 패키지 내의 클래스들과, 상속 관계에 있는 자식 클래스에서 호출 가능
        - default(생략시 기본값): 동이랗ㄴ 패키지 내에 있는 클래스들에서만 호출 가능
    - 반환타입: 메서드가 실행을 마치고 반환할 데이터의 타입
        - `int`, `double`, `String` 등
        - 반환값이 없을 경우 `void`로 명시
    - 메서드명: 메서드의 이름 (식별자 작성규칙을 따름)
    - 매개변수 (parameter): 메서드 호출 시 외부에서 전달받은 값을 저장하는 변수(필요 없을 경우 괄호만 남기고 생략 가능)

- 예시:

```java
public int getSum(int n1, int n2) {
    int sum = n1 + n2;
    return sum;
}
```

#### 2.3 main 메서드

- 자바 프로그램의 시작점 역할을 하는 메소드

- 형식:

```java
// 고전적인 방식
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

- public:
- static:
- void: 반환값 타입
- main: 메서드 이름
- String[] args: 명령행 매개변수, 실행 할 때 사용자가 입력한 값을 args라는 이름의 String 배열에 저장한다.

#### 2.4 메서드 overriding

- 기본적으로 같은 클래스 내에서 메서드의 식별자는 서로 달라야하는게 원칙임
- 단, 메서드의 매개변수가 서로 다른 타입일 경우 식별 가능하기 때문에 같은 이름으로 정의하는게 허용됨
- 메서드를 사용할 때 타입에 관계없이 같은 이름으로 동일한 기능을 수행하도록 할 수 있음
